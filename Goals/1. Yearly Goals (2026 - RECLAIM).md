## 🎯 Yearly Goals by Area

### 💼 Career & Professional Growth (60% of focus)

#### Goal 1: Establish Daily Clarity & Deep Work Protection
**Why:** You lose time to context-switching and unclear priorities. This is foundational.

**Success Criteria:**
- [ ] Morning planning (5 min) identifies ONE focus + project for the day
- [ ] Afternoon review (5 min) captures what shipped, blockers, learnings
- [ ] Weekly project review shows clear progress on netilion2
- [ ] No week drops below 50% deep work time (or documented why)

**Quarterly Breakdown:**
- **Q3:** Establish the ritual; get to 4/5 days/week consistency
- **Q4:** Maintain through busy season; refine as needed
- **Q1 2027:** Assess sustainability

**How to Track:**
- Daily notes: % completion of morning + afternoon reviews
- Weekly plans: document deep work time vs. meetings
- End-of-month: Calculate average deep work %

#### Goal 2: Systematize Learning from Professional Work
**Why:** You do the work but don't capture lessons. Decisions fade. Patterns go unrecognized.**

**Success Criteria:**
- [ ] Each project gets a "Decisions & Learnings" log updated weekly
- [ ] Minimum 8 documented decisions by end of Q3; 20+ by end of year
- [ ] Each decision includes: context, choice, rationale, impact
- [ ] Monthly synthesis: 1 technical + 1 process learning extracted

**Quarterly Breakdown:**
- **Q3:** Establish logging format; capture 8+ decisions from daily work
- **Q4:** Refine format; grow to 12+; start seeing patterns
- **Q1 2027:** Integrate learnings into team practices or personal development

**How to Track:**
- Project CLAUDE.md files: decisions section updated
- Monthly plan: "Key Decisions Made" section filled in
- Quarterly review: count decisions; identify patterns

#### Goal 3: Build Technical Mastery in One Domain
**Why:** Deep expertise makes you more effective and satisfied.

**Success Criteria:**
- [ ] Identify one area: [e.g., architecture patterns, specific tech, performance optimization]
- [ ] Q3: Deep learning phase (reading, experimenting, building)
- [ ] Q4: Applied mastery (teach others, solve hard problem, lead design)
- [ ] By year-end: Can speak authoritatively on this topic to team

**Quarterly Breakdown:**
- **Q3:** Decide domain; consume 3-5 resources (articles, docs, code reviews); build knowledge
- **Q4:** Apply to real project work; document key insights; share with team
- **Q1 2027:** Mentor others or lead initiative in this area

**How to Track:**
- Learning goal in each monthly plan
- Resources consumed documented
- Q4: Can point to decision or project where expertise showed up

#### Goal 4: Clear Project Leadership on netilion2
**Why:** Meaningful work is when you own problems and see them through.

**Success Criteria:**
- [ ] Lead 2-3 meaningful features or initiatives through completion
- [ ] Can describe the impact of your work on netilion2 direction
- [ ] Team sees you as reliable on netilion2 (high code quality, thoughtful reviews)
- [ ] By year-end: Clear on what makes work feel meaningful vs. busywork

**Quarterly Breakdown:**
- **Q3:** Identify meaningful projects; start contribution
- **Q4:** Complete at least 1 major feature; help others understand direction
- **Q1 2027:** Assess: Is netilion2 still meaningful? Any changes needed?

**How to Track:**
- Quarterly plan: features/initiatives led
- Project CLAUDE.md: major work documented
- Team feedback: "You own this"
